# Day 19 — Storytelling & Objection Handling

## STAR Method
- **S**ituation — customer context
- **T**ask — what they needed
- **A**ction — what AWS solution was used
- **R**esult — measurable outcome

### Example STAR Story
- **S** — Retail customer with 50 AWS accounts, no centralized visibility
- **T** — Achieve PCI DSS compliance before Black Friday (8 weeks)
- **A** — Deployed Security Hub with PCI standard, Config rules, GuardDuty across all accounts via Organizations
- **R** — Compliance achieved in 6 weeks, security score 43% → 91%, $2M in fines avoided, $50M in holiday revenue protected

## Communicate at 3 Levels
- **CEO** — business outcome, cost, risk
- **CISO** — security posture, frameworks, tools
- **Security Engineer** — technical architecture, APIs, configs

## Objection Handling
| Objection | Response |
|-----------|----------|
| "AWS is too expensive" | TCO comparison, breach cost, operational savings |
| "On-prem is more secure" | Shared responsibility gap, cloud-scale threats |
| "Don't trust cloud with our data" | KMS, compliance certs, customer key control |
| "Team doesn't know AWS" | Training, ProServe, managed services |
| "We got hacked on AWS" | Analyze root cause, prevent recurrence |
| "Already use [competitor]" | Integration, native advantage |
| "AI too risky" | Bedrock data privacy, Guardrails, no training on your data |
