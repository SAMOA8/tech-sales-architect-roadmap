# Day 17 — Demo Skills & Proof of Concept

## Demo Principles
- Tell a story, not a feature list
- Structure: **Problem → Solution → Value**
- Start with customer pain, end with "what happens next"

### Do's
- Realistic but sanitized data
- Slow down on key moments
- Ask "does this address your concern?" between sections
- Have screenshots as backup

### Don'ts
- Don't demo features you haven't practiced
- Don't read from the screen
- Don't go over time
- Don't show irrelevant features

## Demos to Practice in Your AWS Account
1. **GuardDuty console** — findings dashboard, severity
2. **Security Hub** — security score, compliance standards
3. **CloudTrail** — Event History, find a specific call
4. **IAM** — show a policy document
5. **Your audit tool** — `python audit.py` ← your unique asset!

## 5-Minute Demo Script: Healthcare Customer
- Min 1 — Customer pain (HIPAA fine, breach risk)
- Min 2 — CloudTrail audit trail
- Min 3 — GuardDuty threat detection
- Min 4 — Security Hub compliance score
- Min 5 — Your audit tool (custom security check)
